# Supabase Database Setup for Todos App

To get the todos app working properly, you need to set up the database tables in your Supabase project.

## Create the Todos Table

1. Go to your Supabase project dashboard
2. Navigate to the SQL editor
3. Run the following SQL command to create the todos table:

```sql
create table todos (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users not null,
  title text not null,
  is_complete boolean default false,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable Row Level Security (RLS)
alter table todos enable row level security;

-- Create policies
create policy "Users can insert their own todos" on todos
for insert with check (auth.uid() = user_id);

create policy "Users can view their own todos" on todos
for select using (auth.uid() = user_id);

create policy "Users can update their own todos" on todos
for update using (auth.uid() = user_id);

create policy "Users can delete their own todos" on todos
for delete using (auth.uid() = user_id);
```

## Enable Realtime

1. Go to the Supabase dashboard
2. Navigate to "Database" → "Realtime"
3. Enable Realtime for the "todos" table

## Update Environment Variables

Make sure your `.env` file has the correct Supabase URL and Key:

```
SUPABASE_KEY=your_supabase_anon_key_here
```

Replace `your_supabase_anon_key_here` with your actual Supabase anonymous key from the API settings.

## Testing the Setup

After setting up the database:

1. Run the app with `npm run dev`
2. Open your browser to `http://localhost:5173`
3. Try signing up with an email and password
4. Create some todos to verify everything works correctly
